Web guidelines

(web guidlines will be present in swedish)

    Uppfyllda

  • 1.3.1 - Ange i kod vad sidans olika delar har för roll.

    Har försökt att använda rätt taggar för rubriker, text mm.

  • 1.3.2 - Presentera innehållet i en meningsfull ordning för alla.

    Ordningen är så som jag önskar baserat på skärmstorlek.

  • 1.3.4 - Se till att allt innehåll presenteras rätt oavsett skärmens riktning.

    Innehållet är inte låst till liggande eller stående orientering. Fungerar lika bra oavsett.

  • 1.4.1 - Använd inte enbart färg för att förmedla information.

    Tycker att rubriker/underrubriker mm anger vad som förmedlas.

  • 1.4.10 - Skapa en flexibel layout som fungerar vid förstoring eller liten skärm.

    Layouten fungerar oavsett förstoring eller liten skärm tack vare responsive.

  • 1.4.11 - Använd tillräckliga kontraster i komponenter och grafik.

    • Tycker kontrast mot andra komponenter är tillräckliga för att kunna urskilja vad som är vad.

  • 1.4.13 - Popup-funktioner ska kunna hanteras och stängas av alla.

    Den popup-funktion som finns på sidan kan både stängas på icon X eller genom att klicka på skärmen.

  • 1.4.4 - Se till att text går att förstora utan problem.

    Texten går att förstora med den webbläsare man använder efter önskan och innhehållet går att urskilja.

  • 2.4.7 - Markera tydligt vilket fält eller element som är i fokus.

    Menyval blir förstorad + färgen ändras när muspekaren förs över och när användaren gjort ett val.

  • 2.5.1 - Erbjud alternativ till komplexa fingerrörelser.

    Inte uppfyld till 100%, men bilder på sidan går att förstora med ett klick med mus eller finger på pekskärmar.

  • 2.5.3 - Se till att text på knappar och kontroller överensstämmer med maskinläsbara etiketter.

    Har Aria-label på den knapp jag har som vad jag förstått beskriver funktionen för knappen.

  • 3.1.1 - Ange sidans språk i koden.

    Sidans språk är angett i HTML lang.

  • 3.1.2 - Ange språkförändringar i koden.

    Har angett språkförändringar i HTML lang samt med text på den sida som har annat språk. Då kan automatiskt uppläsning beskriva vilket språk som kommer.

  • 3.2.1 - Ange språkförändringar i koden.

    Har inget oväntat som förändras anser jag.

    Webriktlinjer som ej är uppfyllda

  • 1.3.5 - Märk upp vanliga formulärfält i koden

    Har inget formulärfält

  • 1.4.12 - Se till att det går att öka avstånd mellan tecken, rader, stycken och ord

    Har inte gjort någon funktion för det. Avståndet är anpassat så det ser bra ut på mindre skärmar också. Iaf försökt.

  • 1.4.2 - Ge användaren möjlighet att pausa, stänga av eller sänka ljud.

    Har inte några sånna funktioner.

  • 2.1.1 - Utveckla systemet så att det går att hantera med enbart tangentbordet.

    Har inte haft behov för det.

  • 2.1.2 - Se till att markören inte fastnar vid tangentbordsnavigation

    Har inte någon sådan funktion.

  • 2.1.4 - Skapa kortkommandon med varsamhet

    Har ingte någon sådan funktion.

  • 2.2.2 - Ge användarna möjlighet att justera tidsbegränsningar

    Har ingen tidsbegränsning på sidan.

  • 2.4.1 - Ge användarna möjlighet att pausa eller stänga av rörelser.

    Har inget som behöver pausas.

  • 2.4.1 - Erbjud möjlighet att hoppa förbi återkommande innehåll.

    Har inget återkommande innehåll.

  • 2.4.3 - Erbjud möjlighet att hoppa förbi återkommande innehåll.

    Har inte gett möjlighet att använda tangentbordet.

  • 2.5.2 - Gör det möjligt att ångra klick.

    Har inget som behöver ångras =)

  • 3.2.2 - Utför inga oväntade förändringar vid inmatning.

    Har inget inmatningsfält.

  • 3.3.1 - Utför inga oväntade förändringar vid inmatning.

    Har inget formulär/inmatningsfält.

  • 3.3.2 - Skapa tydliga och klickbara fältetiketter/ledtexter.

    Har inget fält.

  • 3.3.3 - Ge förslag på hur fel kan rättas till.

    Har inget fält eller dylikt.

  • 3.3.4 - Ge möjlighet att ångra, korrigera eller bekräfta vid viktiga transaktioner.

    Har inget fält/inmatning.

  • 4.1.1 - Se till att koden validerar.

    Osäker på vad det betyder, men koden validerar iaf enligt vad som beskrivs för varje section anser jag. Men då jag inte kan allt om validering får den anses som ej uppfylld.

  • 4.1.2 - Se till att skräddarsydda komponenter fungerar i hjälpmedel.

    Har inte kunskap eller haft behov för skräddarsydda komponenter.

  • 4.1.3 - Se till att hjälpmedel kan presentera meddelanden som inte är i fokus.

    Har inte kunskap eller haft behov att anpassa för speciella hjälpmedel för denna sida.
